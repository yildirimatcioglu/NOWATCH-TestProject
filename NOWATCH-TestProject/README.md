# NOWATCH iOS Technical Assignment

Welcome to the NOWATCH iOS technical assignment! This README file provides an overview of the project and outlines the tasks you need to complete. Please follow the instructions carefully and reach out if you have any questions. You have 4 days to complete this assignment.

## Project Overview

In this project, you will work with heart rate data provided in a CSV file. You will parse this data and store it in CoreData. Then, you will display the heart rate data for a selected date on a line chart using Exponential Moving Averages. Additionally, if the selected date is today, you will use a random number generator to show dynamic data for the current day.

## Provided Materials

- **CSV File (heartRate.csv) **: Contains heart rate values on different days.
- **CoreData Implementation**: A basic setup for CoreData is included.
- **Unfinished View**: A view that includes a date picker.
- **Random Number Generator**: For showing dynamic data for the current day.

## Tasks to Complete

### 1. Parse CSV Data and Store in CoreData
- Parse the heart rate data from the provided CSV file.
- Store the parsed data in the CoreData model.

### 2. Display Heart Rate Data on Date Selection
- When a user selects a date using the date picker, retrieve the heart rate data for the selected date from CoreData.
- Display the heart rate data on a line chart using Exponential Moving Averages (EMA). You can use SwiftUI Charts, a 3rd party library, or implement your own chart.

### 3. Dynamic Data for Current Day (Bonus)
- If the user selects today's date, use the random number generator to emit heart rate values.
- Store these values and display them on the line chart using Exponential Moving Averages.
